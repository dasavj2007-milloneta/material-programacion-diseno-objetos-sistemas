Herencia, Constructores y Scanner en Java

Aprenderás a crear una SuperClase que será heredada por tres SubClases diferentes. Dominarás los constructores, el paso de parámetros por valor y referencia, y usarás Scanner para capturar entrada del usuario en IntelliJ IDEA.

1. Estructura del Caso de Estudio

Sistema de Gestión de Empleados de una Empresa

Crearás un sistema para una empresa que gestiona tres tipos de empleados: Ingenieros, Contadores y Vendedores.

El Escenario

📋 Caso: Empresa "TechSolutions"

TechSolutions necesita un sistema para registrar y gestionar información de sus empleados. Cada empleado tiene datos generales (nombre, id, salario base), pero cada tipo de empleado tiene características especiales:

  • Ingeniero: Especialidad técnica, años de experiencia, lenguajes de programación
  • Contador: Número de clientes a cargo, tipo de certificación, auditorías realizadas
  • Vendedor: Zona de ventas, comisión por ventas, meta mensual

ℹ️ Conceptos a Aplicar

  • Herencia: Las 3 subclases extienden de Empleado
  • Constructores: Cada clase inicializa sus atributos
  • Paso por Valor: int, double, boolean (tipos primitivos)
  • Paso por Referencia: String, objetos
  • Scanner: Para capturar datos del usuario

package com.techsolutions.empresa;

public class Empleado {
    // ATRIBUTOS (datos generales de cualquier empleado)
    protected String id;
    protected String nombre;
    protected double salarioBase;
    
    // CONSTRUCTOR - Inicializa los datos del empleado
    public Empleado(String id, String nombre, double salarioBase) {
        this.id = id;
        this.nombre = nombre;
        this.salarioBase = salarioBase;
    }
    
    // GETTERS - Métodos para obtener valores
    public String getId() {
        return id;
    }
    
    public String getNombre() {
        return nombre;
    }
    
    public double getSalarioBase() {
        return salarioBase;
    }
    
    // SETTERS - Métodos para modificar valores
    public void setNombre(String nombre) {
        if (nombre != null && !nombre.isEmpty()) {
            this.nombre = nombre;
        } else {
            System.out.println("El nombre no puede estar vacío");
        }
    }
    
    public void setSalarioBase(double salario) {
        if (salario > 0) {
            this.salarioBase = salario;
        } else {
            System.out.println("El salario debe ser positivo");
        }
    }
    
    // MÉTODOS GENERALES
    
    // Método para calcular el salario (será sobrescrito por subclases)
    public double calcularSalario() {
        return salarioBase;
    }
    
    // Método para mostrar información del empleado
    public void mostrarInfo() {
        System.out.println("════════════════════════════════════");
        System.out.println("📦 Información del Empleado");
        System.out.println("════════════════════════════════════");
        System.out.println("ID: " + id);
        System.out.println("Nombre: " + nombre);
        System.out.printf("Salario Base: $%.2f%n", salarioBase);
        System.out.printf("Salario Total: $%.2f%n", calcularSalario());
        System.out.println("════════════════════════════════════");
    }
}

Explicación del Código

🔍 Entendiendo la SuperClase

  • protected: Los atributos son protected para que las subclases puedan acceder
  • Constructor: Recibe id, nombre y salario. Inicializa todos los atributos
  • Getters/Setters: Permiten acceder y modificar datos de forma controlada
  • calcularSalario(): Método que las subclases sobrescribirán (polimorfismo)
  • mostrarInfo(): Muestra los datos formateados de forma bonita

            this.porcentajeComision = porcentaje;

4. Entendiendo Constructores

Inicialización de Objetos

¿Qué es un Constructor?

Constructor: Método especial que se ejecuta automáticamente cuando creas un objeto. Inicializa los atributos del objeto con valores iniciales. Se asegura que el objeto esté en un estado válido desde el principio.

Características de los Constructores

Mismo nombre que la clase

El constructor debe llamarse exactamente como la clase.

Sin tipo de retorno

No devuelven nada, ni siquiera void.

Se ejecuta automáticamente

Se llama al usar new NombreClase()

Puede haber varios

Sobrecarga: múltiples constructores con diferentes parámetros.

Ejemplo: Flujo de Creación de un Objeto

// Cuando ejecutas esta línea:
Ingeniero ing = new Ingeniero("ENG001", "Juan García", 5000, 
                              "Programación", 5, "Java, Python");

// Ocurre lo siguiente:
// 1. Se reserva espacio en memoria para el nuevo objeto
// 2. Se ejecuta el constructor de Ingeniero con los parámetros dados
// 3. El constructor llama a super() para ejecutar el constructor de Empleado
// 4. Se inicializan todos los atributos
// 5. El objeto está listo para usar

Constructor con super()

📌 Usando super() en SubClases

Cuando una subclase hereda de una superclase, DEBE llamar al constructor del padre usando super():

  • super(parámetros); debe ser la PRIMERA línea del constructor
  • Inicializa los atributos heredados
  • Si no lo escribes, Java lo hace automáticamente (si el padre tiene constructor sin parámetros)

Ejemplo Visual

// SuperClase
public class Empleado {
    protected String id;
    protected String nombre;
    
    public Empleado(String id, String nombre) {
        this.id = id;
        this.nombre = nombre;
        System.out.println("Constructor de Empleado ejecutado");
    }
}

// SubClase
public class Ingeniero extends Empleado {
    private String especialidad;
    
    public Ingeniero(String id, String nombre, String especialidad) {
        super(id, nombre); // Llama al constructor del padre PRIMERO
        this.especialidad = especialidad;
        System.out.println("Constructor de Ingeniero ejecutado");
    }
}

// Uso
Ingeniero ing = new Ingeniero("001", "Juan", "Programación");

// Salida en consola:
// Constructor de Empleado ejecutado
// Constructor de Ingeniero ejecutado

5. Paso de Parámetros por Valor y by Referencia

Diferencias Clave en Java

Tipo 1: Paso por Valor (Tipos Primitivos)

Paso por Valor: Se copia el valor del dato. Los cambios dentro del método NO afectan el original. Aplica a: int, double, boolean, char, etc. 
public class EjemploPasoValor {
    public static void main(String[] args) {
        int edad = 25;
        System.out.println("Edad antes: " + edad); // 25
        
        aumentarEdad(edad); // Pasar por valor
        
        System.out.println("Edad después: " + edad); // Sigue siendo 25
    }
    
    // Este método recibe una COPIA del valor
    public static void aumentarEdad(int edad) {
        edad = edad + 1; // Modifica la COPIA, no el original
        System.out.println("Edad dentro del método: " + edad); // 26
    }
}

// Salida:
// Edad antes: 25
// Edad dentro del método: 26
// Edad después: 25  ← No cambió!

Tipo 2: Paso por Referencia (Objetos y Strings)

Paso por Referencia: Se pasa la dirección (referencia) del objeto, no una copia. Los cambios dentro del método SÍ afectan el original. Aplica a: String, clases, arrays, etc. 
public class EjemploPasoReferencia {
    public static void main(String[] args) {
        Empleado emp = new Empleado("001", "Juan", 5000);
        System.out.println("Nombre antes: " + emp.getNombre()); // Juan
        
        cambiarNombre(emp); // Pasar por referencia
        
        System.out.println("Nombre después: " + emp.getNombre()); // Carlos - ¡CAMBIÓ!
    }
    
    // Este método recibe una REFERENCIA al objeto (no una copia)
    public static void cambiarNombre(Empleado emp) {
        emp.setNombre("Carlos"); // Modifica el OBJETO ORIGINAL
        System.out.println("Nombre dentro del método: " + emp.getNombre()); // Carlos
    }
}

// Salida:
// Nombre antes: Juan
// Nombre dentro del método: Carlos
// Nombre después: Carlos  ← ¡SÍ CAMBIÓ!

Comparación Visual

Aspecto Paso por Valor Paso por Referencia
Tipos int, double, boolean, char String, objetos, arrays
Se copia El valor completo La dirección de memoria
Cambios afectan Solo a la copia local Al objeto original
Eficiencia Más lento (copia datos) Más rápido (copia referencia)
Ejemplo int x = 10; Empleado emp = new Empleado(...);

💡 Consejos Prácticos

  • Usa final para evitar cambios accidentales: final int edad = 25;
  • Para objetos, crea métodos que devuelven parámetros modificados si es necesario
  • Los Strings son especiales: parecen paso por referencia pero se comportan como valor

6. Scanner en Java - Entrada de Datos

Capturando Datos del Usuario en IntelliJ IDEA

¿Qué es Scanner?

Scanner: Clase de Java que permite leer datos del usuario desde la consola. Está en el paquete java.util.Scanner.

Importar Scanner

import java.util.Scanner; // Agregar esta línea al inicio del archivo

Syntax Básico

// Crear un objeto Scanner
Scanner sc = new Scanner(System.in);

// Leer diferentes tipos de datos
String nombre = sc.nextLine();      // Lee una línea completa (con espacios)
int numero = sc.nextInt();          // Lee un número entero
double decimal = sc.nextDouble();   // Lee un número con decimales
boolean boleano = sc.nextBoolean(); // Lee true o false
String palabra = sc.next();         // Lee una palabra (hasta espacio)

// IMPORTANTE: Cerrar Scanner cuando termines
sc.close();

Ejemplo Completo: Leer Datos del Usuario

import java.util.Scanner;

public class LecturaDatos {
    public static void main(String[] args) {
        // Crear un Scanner
        Scanner sc = new Scanner(System.in);
        
        // Pedir datos al usuario
        System.out.print("¿Cuál es tu nombre? ");
        String nombre = sc.nextLine(); // Lee la línea completa
        
        System.out.print("¿Cuántos años tienes? ");
        int edad = sc.nextInt(); // Lee un número entero
        
        System.out.print("¿Cuál es tu salario? ");
        double salario = sc.nextDouble(); // Lee un número con decimales
        
        // Mostrar los datos
        System.out.println("\n═══ DATOS INGRESADOS ═══");
        System.out.println("Nombre: " + nombre);
        System.out.println("Edad: " + edad);
        System.out.printf("Salario: $%.2f%n", salario);
        
        // Cerrar Scanner
        sc.close();
    }
}

⚠️ Problema Común: nextInt() + nextLine()

El Problema

Después de usar nextInt(), nextLine() puede saltarse o no funcionar correctamente.

La Solución

Scanner sc = new Scanner(System.in);

System.out.print("Ingresa tu edad: ");
int edad = sc.nextInt();

sc.nextLine(); // ← IMPORTANTE: Limpiar el buffer

System.out.print("Ingresa tu nombre: ");
String nombre = sc.nextLine(); // Ahora funciona correctamente

sc.close();

Métodos Comunes de Scanner

Método Qué Lee Tipo Ejemplo
nextLine() Una línea completa (con espacios) String "Juan García López"
next() Una palabra (hasta espacio) String "Juan"
nextInt() Un número entero int 25
nextDouble() Un número decimal double 5000.50
nextBoolean() true o false boolean true

ℹ️ Buenas Prácticas

  • Siempre usa nextLine() para leer nombres (que pueden tener espacios)
  • Usa sc.nextLine(); después de números para limpiar el buffer
  • Valida los datos ingresados (que no estén vacíos, etc.)
  • Cierra el Scanner con sc.close(); al final

    static void mostrarTodosEmpleados() {
        if (empleados.isEmpty()) {
            System.out.println("\n❌ No hay empleados registrados");
            return;
        }
        
        System.out.println("\n════════════════════════════════════");
        System.out.println("📋 LISTADO DE TODOS LOS EMPLEADOS");
        System.out.println("════════════════════════════════════");
        
        for (int i = 0; i < empleados.size(); i++) {
            System.out.println("\n[Empleado " + (i + 1) + "]");
            empleados.get(i).mostrarInfo();
        }
    }
    
    // Buscar empleado por ID
    static void buscarEmpleado() {
        System.out.print("\nIngresa el ID del empleado a buscar: ");
        String idBuscado = sc.nextLine();
        
        boolean encontrado = false;
        
        for (Empleado emp : empleados) {
            if (emp.getId().equals(idBuscado)) {
                System.out.println();
                emp.mostrarInfo();
                encontrado = true;
                break;
            }
        }
        
        if (!encontrado) {
            System.out.println("\n❌ Empleado con ID '" + idBuscado + "' no encontrado");
        }
    }
